git报错 remote: Permission to xxx.github.io.git denied to yyy. fatal: unable to access 'https://github.com/yourname/yourname.github.io.git/': The requested URL returned error: 403

发表于 2018-01-20

问题:git报错

remote: Permission to xxx.github.io.git denied to yyy. fatal: unable to access ‘https://github.com/yourname/yourname.github.io.git/‘: The requested URL returned error: 403

解决办法

产生这个问题的背景原因，我的电脑已经配置好一个github的账号，后来想把这个git账号下的工程push到另外一个github账号的仓库中，产生了这个错误。原因是同一台电脑配置的pubkey不能对应两个git账号。

所以需要在配置另个pubkey来给新的git账号

  1. 生成新的pubkey

  2. 将~/.ssh/id_rsa_yyy.pub的内容，添加到git

  3. 修改配置

将config的内容修改为以下内容

Host github.com
HostName github.com
User git
IdentityFile ~/.ssh/id_rsa

Host github-yyy
HostName github.com
User git
IdentityFile ~/.ssh/id_rsa_yyy

4.git remote配置.将remote的链接中git@github.com替换为github-yyy

5.done

问题：fatal: refusing to merge unrelated histories

发表于 2018-01-20

问题：fatal: refusing to merge unrelated histories

解决办法：

git pull origin branchname --allow-unrelated-histories
加上 --allow-unrelated-histories就可以了

说明：

git pull = git fetch + git merge
当进行git pull时,报出以上问题的是git merge 这个步骤。

git merge曾经默认允许合并两个没有共同基础的分支，但是现在已经改成默认不允许，必须加上--allow-unrelated-histories才会进行merge

所以git pull时候加上--allow-unrelated-histories 就能正确运行了

参考：

https://github.com/git/git/blob/master/Documentation/RelNotes/2.9.0.txt#L58-L68

  • “git merge” used to allow merging two branches that have no common
    base by default, which led to a brand new history of an existing
    project created and then get pulled by an unsuspecting maintainer,
    which allowed an unnecessary parallel history merged into the
    existing project. The command has been taught not to allow this by
    default, with an escape hatch “–allow-unrelated-histories” option
    to be used in a rare event that merges histories of two projects
    that started their lives independently.

  • “git pull” has been taught to pass the “–allow-unrelated-histories”
    option to underlying “git merge”.
